# ADR-0006 - UX Principios

## Contexto
La demo debe ser operable y convincente para personal clinico en pocos minutos.

## Decision
- UI en espanol operativo.
- Flujos criticos priorizados: cita -> SOAP -> factura.
- Estados UI consistentes (vacio, carga, error, exito).
- Mensajes de error accionables y uniformes.

## Consecuencias
- Mejor adopcion y menor curva de uso.
- Exige coherencia visual y validaciones de UX continuas.

## Alternativas descartadas
- UI tecnica orientada solo a desarrollador.
- Flujo por modulos aislados sin recorrido end-to-end.
